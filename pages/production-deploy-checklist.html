Production Deploy Checklist (Domain + PM2 + Reverse Proxy + SSL)

Ye checklist PropertySetu ko demo se production-ready deployment me le jane ke liye step-by-step operational flow deti hai, with rollback and security hygiene.

1) Domain & DNS

  • Primary domain set: `propertysetu.in` (example).
  • `A` record server IP par point kare.
  • `www` ko CNAME se primary domain par redirect karo.
  • TTL short rakho during migration (300 sec) and stabilize later.

Before cutover, staging URL par smoke test run karo.

2) App Runtime with PM2

  • Node backend ko PM2 se daemon mode me run karo.
  • Auto-restart + startup script enable karo.
  • Memory limit and log rotation configure karo.
pm2 start server/index.js --name propertysetu-api pm2 save pm2 startup

3) Reverse Proxy (Nginx)

  • Nginx ko public entry point rakho (80/443).
  • Frontend static files serve + `/api` upstream to Node app.
  • Security headers add: `X-Frame-Options`, `X-Content-Type-Options`.
  • Upload size limit and request timeout tune karo.
server { server_name propertysetu.in www.propertysetu.in; root /var/www/propertysetu; location /api/ { proxy_pass http://127.0.0.1:3000/; } }

4) SSL with Let's Encrypt

  • Certbot install karo and Nginx plugin use karo.
  • HTTP to HTTPS force redirect enable karo.
  • Auto-renew cron/systemd timer verify karo.
sudo certbot --nginx -d propertysetu.in -d www.propertysetu.in sudo certbot renew --dry-run

5) Environment & Secrets

  • `.env` me DB URI, JWT secret, storage keys separate rakho.
  • Production me never hardcode secrets in JS/HTML.
  • Separate configs: `dev`, `staging`, `production`.
  • Access control: server SSH keys only, password login disable (recommended).

6) Data Safety & Monitoring

  • Daily DB backup schedule with retention policy.
  • Server metrics: CPU, RAM, Disk alerts.
  • Error logs centralize (PM2 logs + app logs).
  • Incident runbook ready: rollback + DNS failback notes.

7) Release Process

  • `main` se production branch/tag deploy policy define karo.
  • Pre-deploy checklist: migrations, smoke tests, auth check.
  • Post-deploy checklist: login, listing create, media upload, search, care plans.
  • Rollback command + previous release artifact maintain karo.

8) Security Hardening

  • Rate limiting for auth and listing APIs.
  • CORS allowlist strict rakho.
  • File upload validation: MIME + size + extension guard.
  • Audit log for admin actions and sealed bid reveal actions.
